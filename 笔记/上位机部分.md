## 上位机部分

## 基于 MoveIt Servo 的遥控思路

### 一、 核心架构演进思路

当前系统依赖底层手写的 Orocos KDL 状态机进行轨迹规划和正逆运动学解算。为了引入高可靠性的遥控器（D-Bus）控制，并解决冗余自由度（4个输入控制6个轴）带来的**奇异值**和**自干涉/环境碰撞**问题，我们需要将上位机（ROS 2）的核心规划器部分切换至 **MoveIt 2 Servo**。

**工作流对比：**

- **当前流程：** 内部规划轨迹 -> KDL逆解 -> 期望关节角 -> 下发 STM32
- **升级流程（Teleop 模式）：** 遥控器输入 -> 映射为末端 Twist 速度 -> **MoveIt Servo (实时碰撞检测 + DLS 奇异区限速)** -> 安全期望关节角 -> 下发 STM32

------

### 二、 ROS 2 上位机学习路线图 

为了顺利完成此项升级，建议按以下三个阶段进行技术储备：

#### 阶段 1：构建机器人的数字孪生 (MoveIt 2 基础)

- **目标：** 让 ROS 2 知道机械臂的长相、关节极限和周围环境。
- **学习内容：**
  1. 使用 **MoveIt Setup Assistant** 加载您现有的 `first_robot.urdf`，生成 MoveIt 配置包（通常命名为 `first_robot_moveit_config`）。
  2. 学习配置自碰撞矩阵（Self-Collision Matrix），告诉系统哪些连杆相邻可以接触，哪些绝对不能接触。
  3. 学习在 URDF 或通过代码向 `PlanningScene` 中添加静态碰撞体（例如底部的工作台面）。

#### 阶段 2：掌握流式伺服控制 (MoveIt Servo)

- **目标：** 将离散的遥控摇杆数据转化为平滑且安全的连续关节指令。
- **学习内容：**
  1. 研读 **MoveIt Servo 官方文档**，理解其订阅的话题接口（主要是 `geometry_msgs/msg/TwistStamped` 用于笛卡尔速度输入）。
  2. 学习配置 Servo 的 YAML 文件：重点关注 `incoming_command_timeout`（超时保护）、`check_collisions`（开启避障）以及 **奇异点处理参数**（开启基于 DLS 的奇异点规避）。

#### 阶段 3：软硬件联调与手感优化

- **目标：** 让遥控操作丝滑且不卡顿。
- **学习内容：**
  1. 数据滤波：理解如何对 D-Bus 的原始数值设置**死区 (Deadzone)** 和**低通滤波**。
  2. 阻抗融合：研究如何在现有的 `Impdence`（阻抗模式）下叠加 Servo 发送的关节指令，利用底层的柔顺控制吸收急停或轻微碰撞的冲击力。

------

### 三、 上位机代码框架修改建议

代码修改主要集中在通信层的数据解包、控制层的指令映射以及状态机的流转。

#### 1. 通信层：扩充串口解析 (`serial_port.h` & `serial_port.cpp`)

目前您的通信帧主要用于电机数据。需要扩展帧长并定义 D-Bus 数据结构。

**修改点 `serial_port.h`：**

C++

```
// 假设增加了 10 字节用于 D-Bus (4个int16_t通道 + 2个uint8_t拨杆)
#define FRAME_DATA_LENGTH (48 + 10) 

namespace serialport {
    struct DBusData {
        int16_t ch0; // 对应 X 轴速度
        int16_t ch1; // 对应 Y 轴速度
        int16_t ch2; // 对应 Z 轴速度
        int16_t ch3; // 对应 Yaw 轴旋转
        uint8_t sw_l; // 左拨杆 (用于模式切换)
        uint8_t sw_r; // 右拨杆
    };

    class SerialPortWrapper {
    public:
        DBusData getDBusData(); // 暴露 D-Bus 数据接口
    private:
        DBusData dbus_data_;
    };
}
```

**修改点 `serial_port.cpp`：**

在 `parseFrame` 中解包附加在电机数据后面的 D-Bus 数据。

C++

```
bool SerialPortWrapper::parseFrame(const std::vector<uint8_t> &frame, std::vector<uint8_t> &data) {
    // ... CRC 校验保持不变 ...

    // 提取给电机反馈使用的前 48 字节
    data.assign(frame.begin() + 2, frame.begin() + 2 + 48);

    // 提取并拼接 D-Bus 数据
    std::lock_guard<std::mutex> lock(dataMutex);
    int offset = 2 + 48;
    dbus_data_.ch0 = (frame[offset] << 8) | frame[offset + 1];
    dbus_data_.ch1 = (frame[offset + 2] << 8) | frame[offset + 3];
    dbus_data_.ch2 = (frame[offset + 4] << 8) | frame[offset + 5];
    dbus_data_.ch3 = (frame[offset + 6] << 8) | frame[offset + 7];
    dbus_data_.sw_l = frame[offset + 8];
    dbus_data_.sw_r = frame[offset + 9];

    return true;
}
```

#### 3. 主节点层：桥接 D-Bus 与 MoveIt Servo (`light_lift_arm_6dof_node.cpp`)

在这里完成最关键的数据流转换：**串口 D-Bus 数据 -> ROS Twist 消息 -> MoveIt Servo -> ROS Joint 消息 -> 串口下发**。

C++

```
#include <geometry_msgs/msg/twist_stamped.hpp>
#include <sensor_msgs/msg/joint_state.hpp>

class LightLiftArmNode : public rclcpp::Node {
private:
    rclcpp::Publisher<geometry_msgs::msg::TwistStamped>::SharedPtr servo_twist_pub_;
    rclcpp::Subscription<sensor_msgs::msg::JointState>::SharedPtr servo_joint_sub_;
    
    // 定时处理 D-Bus 数据
    void process_dbus_to_servo() {
        auto dbus = serial.getDBusData();
        
        // 1. 物理拨杆切换模式逻辑
        update_mode_from_switches(dbus.sw_l);

        // 如果不是遥控模式，则不发布速度指令
        if (mode_ != State::TeleopServo) return;

        // 2. 将摇杆数值映射为归一化速度 (并带上死区过滤)
        auto apply_deadzone = [](int16_t val, int16_t deadzone) -> double {
            if (abs(val) < deadzone) return 0.0;
            return (val > 0 ? val - deadzone : val + deadzone) / 660.0; // 假设满舵为660
        };

        geometry_msgs::msg::TwistStamped twist_msg;
        twist_msg.header.stamp = this->now();
        twist_msg.header.frame_id = "base_link"; 

        double max_lin_vel = 0.15; // 米/秒
        double max_ang_vel = 0.50; // 弧度/秒

        twist_msg.twist.linear.x = apply_deadzone(dbus.ch0, 20) * max_lin_vel;
        twist_msg.twist.linear.y = apply_deadzone(dbus.ch1, 20) * max_lin_vel;
        twist_msg.twist.linear.z = apply_deadzone(dbus.ch2, 20) * max_lin_vel;
        twist_msg.twist.angular.x = 0.0; // 锁定 Roll
        twist_msg.twist.angular.y = 0.0; // 锁定 Pitch
        twist_msg.twist.angular.z = apply_deadzone(dbus.ch3, 20) * max_ang_vel;

        servo_twist_pub_->publish(twist_msg);
    }

    // 订阅 Servo 规划出的安全关节角
    void init_servo_subscriber() {
        servo_joint_sub_ = this->create_subscription<sensor_msgs::msg::JointState>(
            "/servo_node/target_joint_states", 10,
            [this](const sensor_msgs::msg::JointState::SharedPtr msg) {
                if (mode_ == State::TeleopServo) {
                    // Bypass KDL，直接将 Servo 的安全关节角装载给底层驱动
                    for (size_t i = 0; i < 6; ++i) {
                        motorControl.desir_motor_pos[i] = msg->position[i];
                    }
                }
            });
    }

    void main_control_loop() {
        // 3. 隔离 KDL 计算
        if(mode_ != State::TeleopServo && mode_ != State::JointAutoServo) {
            // 只有非伺服模式时，才走自编的末端轨迹更新和 KDL 逆解
            llarm6dof.update_end_effector_trajectory(realtime);
            llarm6dof.comput_ik(frame);
        }
        
        // 发送给底层 STM32
        motorControl.send_cmd_to_stm32();
    }
};
```
